# 安全开发与运行规范

## 0. 规则定位 `[通用]`

本文件定义 `{PRODUCT_NAME}` 的安全开发、运行和 AI 协作边界。所有涉及认证授权、敏感信息、数据访问、文件上传、对象存储、外部服务、部署发布和私有化交付的变更，均必须遵守本规范。

AI 在执行以下任务前必须先阅读本文件：

- 新增或修改 API、认证、权限、会话、租户隔离能力。
- 新增文件上传、媒体处理、对象存储、导入导出能力。
- 新增数据库、缓存、消息队列、外部服务或第三方 API 集成。
- 修改 `.env.example`、部署脚本、容器镜像、网关、反向代理和发布流程。
- 处理密钥、Token、证书、License、私钥、生产数据或用户敏感数据。

## 1. 文档模块分类 `[通用]`

本模板将安全规则拆分为三类，工程初始化时应根据用户输入生成最终文档：

- `[通用]`：所有项目默认保留的安全基线。
- `[个性化]`：必须根据项目实际技术栈、认证方式、权限模型和数据类型替换的内容。
- `[条件启用]`：仅在项目启用对应能力时保留，例如对象存储、媒体处理、私有化部署、License、SSO、LLM、支付、短信等。

推荐初始化参数：

| 参数 | 用途 |
| --- | --- |
| `{PRODUCT_NAME}` | 项目或产品名称 |
| `{PRODUCT_CODE}` | 项目标识、包名前缀或服务标识 |
| `{AUTH_STRATEGY}` | 认证策略，例如 Session、JWT、OAuth2、OIDC、SSO、API Key |
| `{AUTH_PROVIDER}` | 认证提供方，例如 Keycloak、Auth0、自研 IAM、企业 SSO |
| `{PERMISSION_MODEL}` | 权限模型，例如 RBAC、ABAC、租户隔离、资源可见性策略 |
| `{TOKEN_STORAGE_POLICY}` | 前端 Token 存储策略，例如 HttpOnly Cookie、内存态、受控 localStorage |
| `{SENSITIVE_SECRET_TYPES}` | 需要保护的密钥类型 |
| `{SENSITIVE_DATA_TYPES}` | 需要保护的业务数据和隐私数据类型 |
| `{UPLOAD_SECURITY_POLICY}` | 上传文件类型、大小、扫描和处理策略 |
| `{OBJECT_STORAGE_SECURITY}` | 对象存储桶、访问策略、签名 URL 和生命周期规则 |
| `{EXTERNAL_SERVICE_SECURITY}` | 外部服务调用、超时、重试、脱敏和错误处理策略 |
| `{DEPLOYMENT_SECURITY_BASELINE}` | 部署安全基线，例如 HTTPS、非 root 用户、网关、CORS |
| `{LICENSE_SECURITY_POLICY}` | License、签名、授权校验和私钥管理策略 |
| `{GUARD_POLICY}` | 命令执行、审批、越权操作和高风险变更保护策略 |

## 2. 安全总原则 `[通用]`

- 默认拒绝：未明确允许的访问、上传、导出、管理操作均不得放行。
- 最小权限：用户、服务账号、数据库账号、对象存储账号、容器运行用户只授予完成任务所需的最小权限。
- 后端可信：所有安全判断必须由后端或可信服务执行，前端校验仅用于体验优化。
- 输入不可信：任何来自用户、浏览器、第三方服务、队列、文件、环境变量和模型输出的内容都必须校验。
- 失败安全：认证、权限、License、外部服务、签名校验失败时，应拒绝请求并返回受控错误。
- 可审计：关键安全事件必须记录审计日志，但不得记录密钥、Token、完整凭证或敏感正文。
- 不硬编码：密钥、证书、Token、密码、生产连接串、License 私钥不得写入代码、测试、文档或 Git 历史。
- 不降级：AI 或开发者不得为了通过测试、联调或演示而绕过认证、关闭校验、放宽权限或暴露内部路径。

## 3. 敏感信息与密钥管理 `[通用 + 个性化]`

禁止提交到 Git 的内容包括但不限于：

- `.env`、`.env.local`、`.env.production`、真实部署配置。
- 访问 Token、Refresh Token、Session Secret、JWT Secret、API Key。
- 数据库、缓存、消息队列、对象存储、SMTP、短信、支付、LLM、搜索、监控服务的真实凭证。
- OAuth/OIDC/SSO Client Secret、Webhook Secret、签名密钥、证书私钥。
- License 私钥、授权签名私钥、生产证书、生产连接串。
- 生产数据、用户隐私数据、真实日志、真实导出文件。

文档和样例文件必须遵守：

- `.env.example` 只能包含变量名、格式说明和示例值，不得包含真实值。
- 示例密钥必须使用 `your-xxx-here`、`<REDACTED_DO_NOT_COMMIT>`、`example-secret` 等明显占位符。
- README、部署文档、测试数据、错误示例不得出现真实密钥、真实 Token 或真实用户数据。
- 日志、异常、埋点、审计事件必须对 `{SENSITIVE_SECRET_TYPES}` 和 `{SENSITIVE_DATA_TYPES}` 做脱敏。

## 4. 认证与授权 `[通用 + 个性化]`

项目认证策略为：`{AUTH_STRATEGY}`。认证提供方为：`{AUTH_PROVIDER}`。

通用要求：

- 除显式声明的公开端点外，所有业务 API 默认必须鉴权。
- 公开端点必须在 `rules/api.md`、接口文档和本文件中保持一致，例如健康检查、静态资源、公开文档或登录回调。
- 后端必须从可信认证上下文解析当前用户、租户、角色和权限，不得信任前端传入的 `user_id`、`tenant_id`、`role`、`is_admin`。
- 鉴权失败返回 `401`，权限不足返回 `403`，不得泄露资源是否存在、内部权限规则或系统路径。
- 管理端、开放 API、内部任务、回调接口、Webhook 必须使用不同的认证边界和权限策略。
- 服务间调用必须使用受控凭证、签名、mTLS、内网网关或其他明确机制，禁止裸露内部管理接口。

权限模型为：`{PERMISSION_MODEL}`。生成项目文档时必须明确：

- 角色、权限、资源、租户、组织、数据范围之间的关系。
- 管理端、普通用户端、开放接口、内部接口的访问边界。
- 资源所有者、协作者、只读访问、公开访问和匿名访问的判定方式。
- 权限变更、授权撤销、用户禁用、组织切换后的生效规则。

## 5. 前端安全 `[条件启用]`

当前端存在登录态、管理端、用户端或开放控制台时启用本节。

- Token 存储策略必须遵循 `{TOKEN_STORAGE_POLICY}`，不得随意在 Cookie、localStorage、sessionStorage、URL 参数之间切换。
- 若使用 Cookie，必须根据部署形态配置 `HttpOnly`、`Secure`、`SameSite`、CSRF 防护。
- 若使用前端存储 Token，必须评估 XSS 风险，并避免将 Token 写入日志、错误上报、URL、埋点或截图。
- 前端路由权限仅用于体验优化，后端仍必须进行最终鉴权和授权。
- 前端环境变量只能包含公开配置，禁止放置服务端密钥、对象存储密钥、LLM Key、OAuth Client Secret。
- 401 应进入登录或重新认证流程；403 应展示权限不足，不得自动重试高风险操作。

## 6. 输入校验与 API 安全 `[通用 + 个性化]`

- 请求体、路径参数、查询参数、Header、Cookie、文件元信息均必须通过后端 Schema 或等价机制校验。
- 查询条件必须校验长度、类型、枚举、分页上限、排序字段白名单和过滤条件范围。
- 禁止拼接 SQL、命令、路径、对象存储 Key、模板表达式；数据库查询必须使用参数化机制或 ORM 安全 API。
- 禁止将用户输入直接作为 shell 命令、文件路径、模块名、类名、函数名、URL 跳转目标或模板渲染内容。
- 外部可见错误信息不得包含堆栈、内部路径、连接串、对象 Key、SQL、密钥、Token。
- 对登录、导入导出、上传、短信、邮件、回调、LLM 调用等高成本或高风险接口，应配置限流、幂等、重放保护或审计。

## 7. 文件上传、媒体与对象存储安全 `[条件启用]`

当项目存在上传、导入、媒体处理、对象存储或文件下载能力时启用本节。

上传安全：

- 所有上传必须由后端校验文件大小、扩展名、MIME Type、文件头和业务类型。
- 禁止信任前端传入的文件路径、文件名、MIME Type、对象存储 Key。
- 禁止直接使用用户原始文件名作为存储 Key；应使用服务端生成的随机 Key、业务前缀和安全扩展名。
- 上传失败必须返回受控错误码，不得暴露内部路径、临时目录、对象存储桶结构或服务端栈信息。
- 如项目涉及病毒扫描、内容审核、转码、OCR、解析、解压，应明确超时、大小限制、隔离目录和失败处理。

媒体和文件处理：

- 图片、音视频、压缩包、文档解析不得执行用户可控命令。
- 如必须调用外部程序，应使用参数数组、白名单、沙箱目录、超时和资源限制，禁止 shell 拼接。
- 临时文件必须写入受控目录，并在处理完成、失败或超时后清理。
- 下载和预览必须进行权限校验，不得因为知道 URL 或对象 Key 就能访问。

对象存储：

- `{OBJECT_STORAGE_SECURITY}` 必须明确存储桶权限、访问 Key、签名 URL、生命周期、跨域和公开访问策略。
- 存储桶默认私有；公开读必须有明确业务理由和审查记录。
- 对外访问优先使用短期签名 URL、后端代理或网关鉴权。
- 对象存储账号遵循最小权限，不得使用全局管理员 Key 运行普通业务服务。

## 8. 数据与隐私保护 `[通用 + 个性化]`

项目敏感数据类型为：`{SENSITIVE_DATA_TYPES}`。

- 生产数据不得提交到 Git、测试夹具、示例文档或截图。
- 开发、测试、演示数据必须使用脱敏数据、合成数据或最小化样例。
- 数据导入、导出、下载、报表、备份、日志归档必须进行权限校验和审计。
- 删除、归档、匿名化、保留周期和备份恢复策略应与 `rules/data-management.md` 保持一致。
- 涉及个人信息、商业敏感信息、模型输入输出、交互内容、业务文件正文等数据时，应明确采集目的、访问范围、保存周期和脱敏规则。

## 9. 外部服务与第三方集成 `[条件启用]`

当项目调用 SSO、OAuth、短信、邮件、支付、地图、LLM、语音、OCR、对象存储、Webhook、企业系统或其他第三方服务时启用本节。

- 所有外部调用必须设置超时、错误处理、必要的重试和熔断策略。
- 不得在日志中记录外部服务密钥、Authorization Header、Cookie、签名参数、完整请求正文、Base64 大文件或敏感返回内容。
- 外部服务错误应转换为受控业务错误，不得向客户端透出密钥、供应商内部错误、堆栈或请求签名。
- Webhook 必须验证来源、签名、时间戳或重放保护。
- LLM、语音、OCR 等模型服务不得接收超出业务必要范围的敏感数据；提示词、响应和中间文件的落盘策略必须可控。

## 10. 日志、错误与审计 `[通用]`

- 日志必须默认脱敏 `{SENSITIVE_SECRET_TYPES}` 和 `{SENSITIVE_DATA_TYPES}`。
- 禁止记录明文密码、Token、Cookie、私钥、完整证件号、完整手机号、完整邮箱、本地生产路径、完整对象 Key。
- 生产环境不得向客户端返回堆栈、内部路径、SQL、连接串、第三方服务原始错误。
- 安全相关事件必须可审计，例如登录失败、权限拒绝、管理操作、授权变更、导入导出、批量删除、密钥轮换、License 校验失败。
- 审计日志应记录操作者、租户、资源、动作、结果、时间、来源，但不得记录敏感正文。

## 11. 数据库安全 `[条件启用]`

当项目使用数据库、搜索引擎、缓存或分析仓库时启用本节。

- 数据库账号按环境和服务拆分，遵循最小权限。
- 生产数据库连接串不得出现在 Git、文档、截图和普通日志中。
- SQL 必须参数化；迁移脚本必须经过评审，避免误删、全表更新、不可逆破坏。
- 管理后台、报表、搜索接口必须限制查询范围、分页上限和导出权限。
- 数据库备份、恢复、脱敏、迁移规则应与 `rules/database.md` 和 `rules/data-management.md` 保持一致。

## 12. 部署与容器安全 `[条件启用]`

当项目存在容器、服务器、私有化、Kubernetes、反向代理或网关部署时启用本节。

- `{DEPLOYMENT_SECURITY_BASELINE}` 必须明确 HTTPS、反向代理、CORS、安全 Header、暴露端口和访问边界。
- 容器运行时优先使用非 root 用户，镜像应最小化并避免包含构建密钥、缓存凭证和开发工具。
- 生产环境不得使用默认密码、示例密钥、调试模式、自动生成弱密钥或公开管理端口。
- 反向代理、网关、OpenAPI 文档、监控面板、对象存储控制台必须有访问控制。
- Kubernetes Secret、云厂商密钥、私有化部署凭证不得写入镜像、Helm values 示例或 Git。
- 发布前必须检查 `.env.example`、部署文档、容器镜像、端口映射和对象存储权限是否符合本文件。

## 13. License、私有化与合规适配 `[条件启用]`

当项目存在 License、私有化交付、离线部署、国产化或合规适配时启用本节。

- `{LICENSE_SECURITY_POLICY}` 必须明确授权内容、签名算法、公钥校验、私钥保管、错误码和失败行为。
- License 私钥、签发脚本密钥、客户授权文件原件不得提交 Git。
- 服务端只能保存校验所需的公钥或受控配置，不得内置签发私钥。
- License 校验失败、过期、篡改、超限时必须失败安全，不得自动放行。
- 私有化部署必须明确 HTTPS、网关鉴权、离线镜像、补丁包、配置密钥和客户环境边界。
- 信创、国产数据库、国产中间件、浏览器兼容等要求应同步到 `rules/compatibility.md`。

## 14. Guard 与审批机制 `[条件启用]`

当项目启用 AI Guard、命令审批、变更审批或高风险操作保护时启用本节。

- `{GUARD_POLICY}` 必须明确哪些操作需要人工确认、审批记录和执行边界。
- 删除数据、重置数据库、修改生产配置、执行迁移、调用外部生产服务、跨模块批量改动、仓库外写入文件等操作必须先获得明确授权。
- AI 不得绕过沙箱、审批、权限、测试或安全检查。
- 审批记录建议落入 `docs/approvals/` 或项目约定位置，并与实际变更关联。

## 15. AI 安全行为规则 `[通用]`

AI 生成、修改或执行代码时不得：

- 硬编码密钥、Token、连接串、License 私钥、生产账号。
- 绕过认证、禁用权限检查、伪造当前用户、信任前端传入身份字段。
- 关闭上传校验、放宽对象存储权限、将私有桶改为公开桶。
- 提交真实数据、真实日志、真实导出文件或客户材料。
- 生成危险命令执行逻辑，尤其是由用户输入拼接 shell 命令。
- 为了通过测试而删除安全测试、跳过安全中间件、关闭 CSRF/CORS/鉴权。
- 在未确认的情况下新增公开端点、公开管理后台、公开对象存储桶或公开调试接口。

如安全需求缺失，AI 必须在实现中标记 `待确认`，并优先采用更严格的默认策略。

## 16. 安全变更同步矩阵 `[通用]`

| 变更类型 | 必须同步的文档或产物 |
| --- | --- |
| 认证、权限、会话、租户 | `rules/api.md`、接口文档、OpenAPI、权限说明、测试用例 |
| 密钥、环境变量、配置 | `rules/environment.md`、`.env.example`、部署文档 |
| 上传、媒体、导入导出 | `rules/media.md`、`rules/data-management.md`、测试样例 |
| 对象存储、下载、预览 | `rules/object-storage.md`、部署文档、权限测试 |
| 数据库、迁移、备份 | `rules/database.md`、`rules/data-management.md`、迁移脚本 |
| 外部服务、Webhook、LLM | `rules/api.md`、`rules/environment.md`、错误码和日志脱敏规则 |
| 容器、网关、端口、发布 | `rules/release.md`、`rules/port-management.md`、`rules/compatibility.md` |
| License、私有化、信创 | `rules/release.md`、`rules/compatibility.md`、部署验收清单 |

## 17. 初始化生成建议 `[通用]`

用于工程初始化生成 `security.md` 时，建议按以下步骤处理：

1. 根据用户输入替换 `{PRODUCT_NAME}`、`{PRODUCT_CODE}`、认证策略、权限模型、敏感数据、部署形态等占位符。
2. 保留所有 `[通用]` 章节，并将规则从模板语气改为项目确定性约束。
3. 对 `[个性化]` 章节填入项目真实技术栈和安全策略；信息不足时保留 `待确认`。
4. 对 `[条件启用]` 章节按项目能力裁剪；未启用能力应删除或标记为“不适用”。
5. 与 `global.md`、`api.md`、`environment.md`、`data-management.md`、`database.md`、`media.md`、`object-storage.md`、`release.md` 保持一致。
6. 生成后检查是否仍包含其他项目业务词、真实密钥、真实路径、真实账号或过度宽松的安全描述。

## 18. 完成任务后检查清单 `[通用]`

- 是否没有新增真实密钥、Token、连接串、证书、License 私钥或生产数据。
- 是否所有业务 API 默认鉴权，公开端点有明确理由和文档记录。
- 是否所有权限判断都由后端或可信服务执行。
- 是否所有输入、上传、导入、外部回调都经过校验。
- 是否日志、错误、审计都做了脱敏和受控输出。
- 是否对象存储、下载、预览、导出遵循最小权限。
- 是否部署配置、端口、网关、HTTPS、CORS、安全 Header 已同步到发布文档。
- 是否安全相关变更已补充测试或明确说明验证方式。
