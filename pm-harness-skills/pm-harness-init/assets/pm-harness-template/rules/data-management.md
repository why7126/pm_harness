# 数据管理规范

> 模块标记说明：
>
> - **[通用]**：适用于大多数 Harness 工程，初始化时默认保留。
> - **[个性化]**：必须根据用户项目输入生成，不能直接沿用模板默认值。
> - **[条件启用]**：只有项目具备对应能力时才保留或展开，例如数据库、对象存储、文件上传、导入导出、算法模型、音视频处理。

## 0. 规则定位 [通用]

`rules/data-management.md` 约束项目中所有数据资产的存放位置、提交边界、脱敏要求、运行时隔离和 AI 修改规则。

AI Agent 在以下场景必须读取本文件：

- 新增或修改 `data/`、`tests/fixtures/`、`models/`。
- 新增数据库 schema、seed、migration、fixtures。
- 新增文件上传、图片/音频/视频/文档管理。
- 新增导入导出、备份恢复、数据初始化。
- 新增算法模型、模型权重、训练数据、推理样例。
- 处理日志、缓存、运行时数据库、临时文件。

初始化生成本文件时，必须根据项目实际数据类型替换占位符；未知项标记为 `待确认`，不得编造数据来源、许可或脱敏状态。

常用占位符：

| 占位符 | 含义 | 归属 |
|---|---|---|
| `{PRODUCT_NAME}` | 产品或项目名称 | [个性化] |
| `{DATA_ASSET_MATRIX}` | 数据资产目录矩阵 | [个性化] |
| `{SENSITIVE_DATA_TYPES}` | 敏感数据类型 | [个性化] |
| `{DATABASE_STACK}` | 数据库与 schema/seed 策略 | [条件启用] |
| `{OBJECT_STORAGE_STACK}` | 对象存储与上传文件策略 | [条件启用] |
| `{IMPORT_EXPORT_TYPES}` | 导入导出文件类型 | [条件启用] |
| `{MODEL_ASSET_POLICY}` | 模型文件与训练数据策略 | [条件启用] |
| `{DATA_RETENTION_POLICY}` | 数据保留与清理策略 | [条件启用] |

## 1. 数据资产目录矩阵 [通用 + 个性化]

项目数据资产必须有明确位置和提交边界。

初始化时生成本项目矩阵：

```text
{DATA_ASSET_MATRIX}
```

推荐矩阵：

| 目录/位置 | 用途 | 是否提交 Git | 说明 |
|---|---|---:|---|
| `data/README.md` | 数据目录说明 | 是 | 说明目录结构和提交边界 |
| `data/samples/` | 脱敏样例数据 | 视情况 | 小体积、可公开、已脱敏 |
| `data/runtime/` | 本地运行时数据 | 否 | 数据库、缓存、运行文件 |
| `data/uploads/` | 本地上传文件 | 否 | 图片、音视频、文档 |
| `data/processed/` | 本地处理结果 | 否 | 转码、解析、中间产物 |
| `data/tmp/` | 临时文件 | 否 | 可随时删除 |
| `tests/fixtures/` | 测试 fixtures | 是 | 必须脱敏、可公开 |
| `src/**/schema*` | schema/DDL | 是 | 数据库结构事实源 |
| `src/**/seed*` | 种子数据 | 视情况 | 不得包含真实用户数据 |
| `models/` | 模型说明、校验和、下载指引 | 视情况 | 大权重默认不提交 |
| 对象存储 | 运行时上传和导出文件 | 不适用 | 不进入 Git |
| `logs/` | 运行日志 | 否 | 默认不提交 |

## 2. 可提交内容 [通用]

允许提交：

- `.gitkeep`。
- 数据目录说明文档。
- 数据 schema、DDL、migration。
- 不含真实用户信息的 seed 数据。
- 脱敏测试 fixtures。
- 小体积、可公开、来源清晰的样例数据。
- 示例配置中的占位数据。
- 模型下载说明、版本、校验和、许可说明。

提交前必须确认：

- 数据来源合法。
- 已脱敏。
- 文件大小适合 Git 管理。
- 不含密钥、token、真实身份信息或生产业务数据。
- 有明确用途说明。

## 3. 禁止提交内容 [通用 + 个性化]

禁止提交：

- 真实用户数据、客户数据、业务交易数据。
- 真实图片、音频、视频、文档、会议记录、声纹、身份证件、地址、手机号、邮箱等敏感内容。
- 运行时数据库文件，例如 SQLite `.db`、PostgreSQL/MySQL 数据目录。
- 对象存储运行时文件。
- 上传目录、处理结果、缓存、临时文件。
- 生产日志、调试日志、执行轨迹。
- `.env`、真实凭据、连接串、token、私钥。
- 未确认许可的大模型权重、商业模型、训练数据集。

项目敏感数据类型：

```text
{SENSITIVE_DATA_TYPES}
```

如不确定某数据是否可提交，默认不可提交。

## 4. data 目录规则 [通用]

`data/` 用于本地开发、演示、测试样例和运行时数据承载，不作为业务源码目录。

推荐结构：

```text
data/
├── README.md
├── samples/
├── runtime/
├── uploads/
├── processed/
└── tmp/
```

规则：

- `data/runtime/`、`data/uploads/`、`data/processed/`、`data/tmp/` 默认不提交 Git。
- `data/samples/` 只有在小体积、脱敏、可公开时才可提交。
- 新增子目录必须更新 `data/README.md` 和 `.gitignore`。
- 不得把业务源码放入 `data/`。

## 5. 样例数据规则 [通用]

样例数据必须满足：

- 可公开。
- 已脱敏。
- 不包含真实用户、客户、生产环境、商业秘密。
- 文件大小适合 Git 管理。
- 有明确用途说明。
- 可由测试或演示重复使用。

推荐样例命名：

```text
sample-<domain>-<scenario>.<ext>
demo-<scenario>.<ext>
```

不得使用真实客户名称、真实手机号、真实地址、真实订单、真实会议标题作为样例。

## 6. 测试数据与 fixtures [通用]

测试数据应放入：

```text
tests/fixtures/
```

规则：

- 测试 fixtures 必须脱敏、可公开。
- 测试中不得依赖真实外部服务或生产数据。
- 推荐使用 factory、builder 或 fixture 函数生成测试数据。
- 测试执行后不得污染本地运行时数据。
- 大型测试文件应使用生成脚本或下载说明，不直接提交。

新增或修改测试数据时，应同步：

```text
tests/fixtures/README.md（如存在）
rules/testing.md
```

## 7. 数据库 schema、migration 与 seed [条件启用]

数据库技术栈：

```text
{DATABASE_STACK}
```

规则：

- schema、migration、ORM model、Repository、测试数据必须同步。
- seed 数据不得包含真实用户信息。
- 多数据库项目必须保持各数据库 DDL 语义一致。
- 修改 schema 时必须补充迁移、回滚说明和测试。
- 本地运行时数据库文件不得提交。

种子数据必须说明：

- 用途。
- 执行方式。
- 是否可重复执行。
- 是否影响已有数据。
- 如何在已有环境中手动应用。

## 8. 文件上传、媒体与对象存储数据 [条件启用]

对象存储策略：

```text
{OBJECT_STORAGE_STACK}
```

涉及文件、图片、音频、视频、文档上传时：

- 上传文件进入对象存储或运行时上传目录，不进入 Git。
- 必须校验 MIME、大小、权限和内容边界。
- 元数据写入数据库，文件本体写入对象存储或文件系统。
- 预览 URL、下载 URL、对象 key 不得泄露未授权资源。
- 本地上传目录必须加入 `.gitignore`。

媒体处理产物，例如缩略图、转码文件、字幕、解析中间文件，应放入运行时或对象存储，不直接提交。

## 9. 导入导出、备份恢复数据 [条件启用]

导入导出类型：

```text
{IMPORT_EXPORT_TYPES}
```

规则：

- 导入模板可以提交，但必须使用假数据。
- 导出文件属于运行时数据，默认不提交。
- 备份文件默认不提交，尤其不能提交生产备份。
- 导入失败明细、临时解析文件、处理日志默认不提交。
- 导入导出涉及字段变更时，必须同步 API、文档、测试和验收标准。

## 10. 模型文件、训练数据与算法样例 [条件启用]

模型资产策略：

```text
{MODEL_ASSET_POLICY}
```

规则：

- 大模型权重、商业模型、真实训练数据默认不提交 Git。
- `models/` 可放置 README、版本说明、校验和、下载脚本或小体积 dummy 模型。
- 模型文件必须说明来源、许可、版本、校验和、放置路径。
- 推理样例必须脱敏、可公开、体积可控。
- 训练数据、声纹、音频、人脸、医疗、金融等敏感数据不得提交。
- Docker 或部署环境应通过 volume、对象存储或制品仓库挂载模型。

## 11. 日志、缓存与执行轨迹 [通用]

禁止提交：

- `logs/` 中的运行日志。
- `*.log`。
- `*.jsonl` 执行轨迹。
- 缓存目录。
- 临时下载目录。
- 构建产物和测试临时产物。

如果日志需要作为 BUG 证据，应：

- 脱敏。
- 截取必要片段。
- 放入 `issues/bugs/<BUG-ID>/logs/`。
- 删除 token、用户信息、路径敏感信息。

## 12. 脱敏规则 [通用 + 个性化]

脱敏对象包括但不限于：

- 姓名、手机号、邮箱、身份证件、地址。
- 用户 ID、租户 ID、订单号、合同号、会议号。
- 图片、音频、视频、声纹、人脸、定位。
- 访问 token、密钥、连接串、内部 URL。
- 价格协议、客户清单、商业数据。

脱敏要求：

- 保留数据结构，不保留真实值。
- 使用明显假数据，例如 `Test User`、`demo@example.com`、`13800000000`。
- 不使用真实生产 ID 做“看起来无害”的样例。
- 脱敏后仍需检查元数据，例如图片 EXIF、文档作者、音视频 metadata。

## 13. .gitignore 与提交边界 [通用]

新增数据目录、运行时文件类型或导入导出能力时，必须检查 `.gitignore`。

推荐忽略：

```text
data/runtime/
data/uploads/
data/processed/
data/tmp/
logs/
*.log
*.jsonl
*.db
*.sqlite
*.sqlite3
*.bak
*.dump
*.tmp
```

具体规则应根据项目技术栈扩展。

## 14. AI 生成/更新规则 [通用]

AI 如果新增以下能力：

- 文件上传。
- 图片、音频、视频、文档管理。
- Excel/CSV/JSON 导入导出。
- 数据备份恢复。
- 样例数据生成。
- 数据库初始化脚本。
- 模型文件或推理样例。
- 运行日志、缓存、执行轨迹。

必须同步检查并更新：

```text
data/README.md
.gitignore
.env.example
docs/04-database-design.md
docs/07-object-storage-strategy.md（如涉及对象存储）
rules/data-management.md
rules/security.md
rules/media.md（如涉及媒体）
rules/object-storage.md（如涉及对象存储）
tests/fixtures/
```

## 15. 完成任务后检查清单 [通用 + 条件启用]

```text
□ 是否确认新增/修改数据属于哪类资产
□ 是否确认该数据是否允许提交 Git
□ 是否完成脱敏并检查元数据
□ 是否更新 data/README.md
□ 是否更新 .gitignore
□ 是否未提交运行时数据库、上传文件、日志、缓存、临时文件
□ 数据库变更：是否同步 schema、migration、seed、fixtures
□ 文件/媒体变更：是否同步对象存储策略和权限规则
□ 导入导出变更：是否同步模板、失败明细和测试数据
□ 模型/算法变更：是否说明模型来源、许可、版本、校验和和放置路径
□ 是否补充 tests/fixtures 或生成脚本
```

## 16. 初始化生成建议 [通用]

工程初始化工具生成 `rules/data-management.md` 时应：

1. 保留 [通用] 模块。
2. 用用户输入替换 [个性化] 占位符。
3. 根据项目能力保留或删除 [条件启用] 模块。
4. 根据产品领域生成敏感数据类型列表。
5. 根据数据库、对象存储、导入导出、算法模型能力生成数据资产矩阵。
6. 未知的数据来源、许可、保留周期标记为 `待确认`。
7. 删除指向不存在目录、文档或能力的内容。
8. 保持 data-management.md 与 security.md、media.md、object-storage.md、database.md、testing.md、directory-structure.md 一致。
