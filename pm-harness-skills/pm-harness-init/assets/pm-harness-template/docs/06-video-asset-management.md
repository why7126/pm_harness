# 视频与富媒体资产管理说明

## 0. 文档定位 `[通用]`

本文档用于定义项目中视频与富媒体资产的管理规则，覆盖从需求进入、上传接入、对象存储、数据库元数据、封面生成、转码处理、播放展示、权限控制、测试验收到后续治理的完整链路。

本文档可以作为工程初始化模板使用。初始化时应根据用户输入判断是否启用视频能力，并替换所有 `{...}` 占位符。

## 2. 启用条件 `[通用]`

视频能力默认属于条件启用模块。满足以下任一条件时，应生成完整视频资产管理方案：

- 产品需求中存在视频上传、播放、预览、回放、素材管理、内容审核、转码或分发能力。
- 产品形态包含面向用户的视频播放端，或包含面向运营/管理员的视频维护端。
- 数据模型中存在媒体、附件、素材、课程、商品、内容、事件、回放等需要绑定视频资产的对象。
- 系统需要统一治理图片、音频、视频等富媒体资产。

未启用视频能力时，应在本文档中保留“未启用说明”和“未来启用条件”，删除强制实现、测试和部署要求。

常见场景参考：

- 用户上传的视频素材。

- 审核前后的媒体资产管理。
- 面向移动端或小程序的轻量化播放。

## 5. 视频资产生命周期 `[通用]`

```text
选择/录制视频
  -> 客户端预校验
  -> 获取上传凭证或走后端上传
  -> 上传原始文件
  -> 记录媒体元数据
  -> 生成或上传封面
  -> 可选：异步转码/压缩/截图
  -> 可选：内容审核
  -> 发布或绑定业务对象
  -> 播放与分发
  -> 下架、归档或删除
```

每个阶段都必须有明确状态，避免只有对象存储文件但没有业务元数据，或只有数据库记录但对象不存在。

## 6. 上传策略 `[通用 + 个性化]`

| 项 | 规则 |
|---|---|
| 上传入口 | `{UPLOAD_ENTRYPOINT}` |
| 上传方式 | 后端中转 / 预签名直传 / 分片上传 / 待确认 |
| 最大文件大小 | `{UPLOAD_MAX_SIZE}` |
| 允许格式 | `{VIDEO_FORMATS}` |
| MIME 校验 | 必须校验扩展名、MIME、文件头，不得只信任客户端传值 |
| 重复上传 | 按业务需要启用 hash 去重 |
| 失败处理 | 上传失败不得生成可播放资产 |
| 临时文件 | 必须有过期清理策略 |

大文件或移动端弱网场景应优先考虑分片上传、断点续传、进度展示和失败重试。

## 7. 对象存储设计 `[条件启用 + 个性化]`

当 `{OBJECT_STORAGE_STACK}` 不为空时启用本节。

### 7.1 存储容器

| 类型 | bucket / container | 访问级别 | 说明 |
|---|---|---|---|
| 原始视频 | `{VIDEO_BUCKET}` | 私有 / 公开 / 待确认 | 保存上传原文件 |
| 转码视频 | `{TRANSCODED_VIDEO_BUCKET}` | 私有 / 公开 / 待确认 | 启用转码时生成 |
| 视频封面 | `{VIDEO_COVER_BUCKET}` | 私有 / 公开 / 待确认 | 保存上传或截帧封面 |
| 临时文件 | `{TEMP_MEDIA_BUCKET}` | 私有 | 上传、审核或转码中间文件 |

### 7.2 Key 命名规则

```text
source: {VIDEO_KEY_PATTERN}
cover:  {COVER_KEY_PATTERN}
transcoded: {TRANSCODED_KEY_PATTERN}
temp:   {TEMP_MEDIA_KEY_PATTERN}
```

命名规则要求：

- 必须包含业务对象、业务对象 ID、媒体 ID 或版本号中的必要字段。
- 不得使用原始文件名作为唯一 key。
- 删除或归档策略必须与数据库状态一致。
- 公开访问 URL、签名 URL、CDN URL 的生成规则必须与 `rules/object-storage.md` 一致。

## 8. 数据库元数据 `[通用 + 个性化]`

视频资产必须有数据库元数据记录，推荐归入统一媒体表或业务专属媒体表。

| 字段 | 说明 | 是否必需 |
|---|---|---|
| `id` | 媒体 ID | 是 |
| `business_type` | 绑定业务类型 | 按需 |
| `business_id` | 绑定业务对象 ID | 按需 |
| `media_type` | 媒体类型，视频建议为 `video` | 是 |
| `status` | 上传中、可用、转码中、失败、下架、删除等 | 是 |
| `source_object_key` | 原视频对象 key | 是 |
| `cover_object_key` | 封面对象 key | 按需 |
| `transcoded_object_key` | 转码后对象 key | 按需 |
| `duration` | 视频时长 | 按需 |
| `width` / `height` | 视频尺寸 | 按需 |
| `mime_type` | MIME 类型 | 是 |
| `file_size` | 文件大小 | 是 |
| `checksum` | 文件摘要 | 按需 |
| `sort_order` | 排序 | 按需 |
| `created_by` / `updated_by` | 操作人 | 按需 |
| `created_at` / `updated_at` | 时间戳 | 是 |

生成时应将 `{MEDIA_TABLE}` 与 `docs/04-database-design.md` 的表结构保持一致。

## 9. 封面策略 `[条件启用]`

封面能力按项目需求选择：

- 手动上传封面。
- 自动截取首帧。
- 指定时间点截帧。
- 从视频中选择帧作为封面。
- 使用默认封面占位图。

封面要求：

- 封面尺寸、格式、大小限制必须明确。
- 封面缺失时必须有降级展示策略。
- 封面对象的生命周期必须跟随视频资产。
- 自动截帧属于异步任务时，应记录任务状态和失败原因。

## 10. 转码与压缩 `[条件启用]`

当 `{TRANSCODING_ENABLED}` 为 true 时启用本节。

| 项 | 规则 |
|---|---|
| 转码服务 | `{TRANSCODING_PROVIDER}` |
| 触发方式 | 上传后自动 / 手动触发 / 定时任务 / 待确认 |
| 输出格式 | `{TRANSCODED_FORMATS}` |
| 清晰度 | 原画 / 1080p / 720p / 480p / 待确认 |
| 失败重试 | `{TRANSCODING_RETRY_POLICY}` |
| 状态回写 | 必须更新媒体元数据状态 |
| 原文件保留 | 保留 / 删除 / 归档 / 待确认 |

转码实现应优先放入异步任务或独立服务，避免阻塞 API 请求。

## 11. 播放与分发 `[通用 + 个性化]`

播放策略应根据端能力和访问控制决定：

| 场景 | URL 策略 | 缓存策略 | 权限要求 |
|---|---|---|---|
| 公开展示 | CDN URL / 公共对象 URL | 可长缓存 | 无或弱权限 |
| 登录后播放 | 签名 URL / 后端鉴权代理 | 短缓存 | 需要登录 |
| 管理后台预览 | 签名 URL / 内部代理 | 短缓存 | 管理权限 |
| 私有素材 | 短期签名 URL | 不长期缓存 | 严格鉴权 |

要求：

- 私有视频不得直接暴露永久公开 URL。
- 播放失败时应区分文件不存在、权限不足、格式不支持、转码未完成。
- 启用 CDN 时，缓存刷新和下架策略必须明确。

## 12. 安全与合规 `[通用]`

- 上传接口必须校验登录态、权限、文件大小、格式和业务对象归属。
- 不得信任客户端提供的文件名、MIME、大小、时长、封面路径。
- 私有视频必须使用鉴权 URL、签名 URL 或后端代理。
- 删除视频时应区分软删除、硬删除、归档和对象清理。
- 涉及用户上传内容时，应预留内容审核、举报、下架和审计能力。
- 日志不得记录长期有效的私有访问 URL 或敏感签名参数。

## 13. API 约定 `[条件启用 + 个性化]`

视频 API 必须与 `docs/03-api-index.md` 和 `rules/api.md` 保持一致。

| 能力 | API 示例 | 状态 |
|---|---|---|
| 创建上传任务 | `POST {API_PREFIX}/media/videos/uploads` | 待确认 |
| 完成上传 | `POST {API_PREFIX}/media/videos/{id}/complete` | 待确认 |
| 更新封面 | `PUT {API_PREFIX}/media/videos/{id}/cover` | 待确认 |
| 查询转码状态 | `GET {API_PREFIX}/media/videos/{id}/transcoding` | 待确认 |
| 获取播放地址 | `GET {API_PREFIX}/media/videos/{id}/play-url` | 待确认 |
| 删除视频 | `DELETE {API_PREFIX}/media/videos/{id}` | 待确认 |

初始化时应按实际业务对象生成 API，不得保留无业务来源的固定路径。

## 14. 前端体验要求 `[条件启用]`

启用前端视频能力时，应覆盖：

- 上传前校验、上传进度、取消上传、失败重试。
- 封面预览、视频预览、播放失败提示。
- 转码中、审核中、已下架、已删除等状态展示。
- 移动端弱网和小屏适配。
- 无障碍要求，包括可识别的播放控件、焦点状态和替代文本。

前端页面和组件必须与 `rules/ui-design.md`、`docs/05-compatibility-matrix.md` 保持一致。

## 15. 测试与验收 `[通用 + 个性化]`

| 测试类型 | 验证内容 | 是否必需 |
|---|---|---|
| 单元测试 | 文件校验、key 生成、状态流转 | 是 |
| API 测试 | 上传、完成、查询、删除、权限 | 是 |
| 集成测试 | 对象存储、数据库、异步任务 | 条件启用 |
| 前端测试 | 上传交互、预览、错误提示 | 条件启用 |
| 兼容性测试 | 浏览器、移动端、小程序、格式支持 | 条件启用 |
| 安全测试 | 越权访问、非法格式、超大文件、签名 URL | 是 |

验收标准：

- 成功上传后，数据库元数据与对象存储文件一致。
- 删除、下架或替换视频后，播放入口状态正确。
- 转码失败、封面失败、对象不存在等异常有可理解提示。
- 权限不足用户无法上传、查看或删除不属于自己的视频。
- `{VIDEO_TEST_COMMAND}` 可以覆盖项目声明的关键验证路径；无法自动化时必须补充人工验证步骤。

## 16. 初始化阶段建议 `[通用]`

视频能力可按阶段接入：

1. 基础阶段：定义目录、元数据表、对象存储 key 规则和上传接口占位。
2. MVP 阶段：支持视频上传、封面上传、播放预览、删除和基础权限。
3. 增强阶段：支持自动封面、转码、压缩、多清晰度、CDN 分发。
4. 治理阶段：支持内容审核、归档、生命周期清理、监控告警和成本统计。

如果项目当前不需要视频能力，应保留本文档作为未来启用规范，但在初始化输出中标记 `{VIDEO_ENABLED}=false`。

```text

```

修改要求：

- 新增视频能力时，不得只改前端或只改接口，必须同步数据、存储、安全和测试规则。
- 修改对象 key、bucket、访问策略或 URL 签名规则时，必须同步 `rules/object-storage.md`。
- 修改上传大小、格式、转码或封面策略时，必须同步测试用例和验收标准。
- 删除视频能力时，必须说明迁移、历史数据和对象清理策略。

## 18. 更新触发条件 `[通用]`

以下变化必须更新本文档：

- 新增或删除视频业务场景。
- 新增端形态或端能力变化。
- 上传方式、大小限制、格式白名单变化。
- 对象存储、bucket、key 命名、CDN 或签名 URL 策略变化。
- 数据库媒体表、字段、状态机或绑定关系变化。
- 封面、转码、审核、归档、清理策略变化。
- 视频相关 API、权限、测试或验收规则变化。
