# 安全开发规范

## 1. 认证与权限

- 管理端仅供企业内部员工使用，必须与店主展示端区分权限。
- 店主端只能查看允许公开的瓷砖信息。
- 管理端维护瓷砖信息、图片、视频、上下架状态时必须鉴权。

## 2. 文件上传安全

- 所有上传必须经过后端校验。
- 必须限制文件大小、MIME Type、扩展名。
- 禁止使用用户原始文件名作为对象存储Key。
- 禁止信任前端传入的文件路径。
- 上传失败必须返回明确错误码，不得暴露内部路径。

## 3. 视频安全

- 视频上传必须限制大小和类型。
- 视频封面生成和转码任务不得执行用户可控命令。
- 临时文件必须写入 `data/tmp/`，处理完成后清理。

## 4. MinIO安全

- 真实AccessKey和SecretKey不得提交Git。
- `.env.example` 只能放示例值。
- 对外访问优先使用签名URL。
- 存储桶权限必须遵循最小权限原则。

## 5. 输入校验

- API请求体必须使用Pydantic Schema。
- 查询条件必须做长度、类型、枚举校验。
- SQL查询必须参数化。

## 6. AI安全要求

AI生成代码时不得：

- 硬编码密钥。
- 绕过鉴权。
- 关闭上传校验。
- 提交真实数据文件。
- 生成危险命令执行逻辑。
