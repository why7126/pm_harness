# 安全开发规范

## 1. 认证与权限

- 管理端、公开端、移动端、内部接口必须区分权限边界。
- 任何写操作、上传、删除、导出、后台任务触发必须鉴权。
- 权限策略变化必须进入 OpenSpec Change，并补充测试。

## 2. 文件上传安全

- 所有上传必须经过后端校验。
- 必须限制文件大小、MIME Type、扩展名。
- 禁止使用用户原始文件名作为对象 Key。
- 禁止信任前端传入的文件路径。
- 上传失败不得暴露内部路径、密钥或完整堆栈。

## 3. 对象存储安全

- 真实 AccessKey、SecretKey、Token 不得提交 Git。
- `.env.example` 只能放示例值。
- 对外访问优先使用签名 URL 或受控公开策略。
- 存储桶和对象前缀遵循最小权限原则。

## 4. 输入与数据安全

- API 请求体必须使用 Schema 校验。
- 查询条件必须做长度、类型、枚举和权限校验。
- SQL 查询必须参数化。
- 日志、导出、测试数据不得包含真实用户或客户敏感信息。

## 5. AI 禁止行为

AI 生成代码时不得：

```text
□ 硬编码密钥
□ 绕过鉴权
□ 关闭上传校验
□ 提交真实数据文件
□ 生成危险命令执行逻辑
□ 把生产域名、连接串或客户数据写进文档示例
```
