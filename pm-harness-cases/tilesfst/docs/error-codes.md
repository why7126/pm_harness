# 错误码体系

## 分段规则

| 区间 | 含义 |
|------|------|
| `0` | 成功 |
| `1xxxx` | 系统错误 |
| `2xxxx` | 认证授权 |
| `3xxxx` | 业务错误 |
| `4xxxx` | 参数错误 |
| `5xxxx` | 外部依赖（MinIO、存储等） |

## 已登记错误码

| 码 | HTTP | 说明 |
|----|------|------|
| 0 | 200 | 成功 |
| 10001 | 500 | 系统内部错误 |
| 20001 | 401 | 未登录或 token 无效 |
| 20002 | 401 | 登录凭证错误 |
| 20003 | 403 | 账号已停用 |
| 20004 | 403 | 无权限 |
| 30001 | 404 | 瓷砖不存在 |
| 30002 | 409 | 瓷砖名称冲突 |
| 40001 | 400 | 请求参数无效 |
| 40002 | 400 | 分页参数无效 |
| 50001 | 502 | 对象存储不可用 |
| 50002 | 400 | 文件类型不允许 |
| 50003 | 400 | 文件大小超限 |

## 实现

- Python 常量：`src/backend/app/core/error_codes.py`
- 异常：`src/backend/app/core/exceptions.py`

## AI 规则

新增业务错误 MUST 在本表与 `error_codes.py` 同步登记，禁止随意使用未文档化数字。
