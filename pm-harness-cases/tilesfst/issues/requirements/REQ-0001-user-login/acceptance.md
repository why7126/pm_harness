# 验收标准

## 1. 功能验收

### 1.1 登录页访问与展示

- [ ] 用户可通过 `/admin/login` 打开 Web 管理端登录页。
- [ ] 登录页采用左右分屏布局：左侧品牌视觉区、右侧登录表单区（桌面端）。
- [ ] 移动端（`< 1024px`）隐藏左侧视觉区，登录表单全屏居中展示。
- [ ] 页面视觉与 `prototype/web/user-login.png` 一致，遵循暗色工业风（底色 `#18160F`、品牌金 `#C8A055`）。
- [ ] 登录表单包含：用户名、密码、记住我、忘记密码、登录按钮、企业微信入口、语言切换占位、页脚版权。

### 1.2 表单校验与提交

- [ ] 用户名为空时不能提交，并提示「请输入用户名」。
- [ ] 密码为空时不能提交，并提示「请输入密码」。
- [ ] 点击登录后按钮进入 loading 状态，防止重复提交。
- [ ] 密码输入框支持显隐切换（眼睛图标）。
- [ ] 用户名或密码输入框内按 Enter 可提交登录。

### 1.3 登录成功与失败

- [ ] 正确账号密码可登录成功，并跳转 `/admin/dashboard`。
- [ ] 错误账号或密码不能登录，展示统一提示「账号或密码错误」，不清空账号。
- [ ] 被禁用账号不能登录，提示「账号已停用，请联系管理员」。
- [ ] 登录成功后刷新页面仍保持登录状态（记住我勾选时延长有效期）。

### 1.4 角色与权限

- [ ] 登录成功后后端返回用户 ID、显示名、角色、状态。
- [ ] `admin`、`employee` 角色可进入管理端受保护页面。
- [ ] `store_owner` 角色尝试访问管理端受保护页面时被拒绝，展示无权限提示或跳转无权限页。
- [ ] 非管理员访问管理员专属页面时被拒绝。

### 1.5 路由守卫

- [ ] 管理端除 `/admin/login` 外均为受保护路由。
- [ ] 未登录访问受保护路由时自动跳转 `/admin/login`。
- [ ] 已登录访问 `/admin/login` 时自动跳转管理端首页。
- [ ] Token 过期时跳转登录页，并提示「登录已过期，请重新登录」。

### 1.6 退出登录

- [ ] 用户可在管理端顶部菜单退出登录。
- [ ] 退出后清除本地登录态并跳转登录页。
- [ ] 已退出用户再次访问管理端受保护页面时跳转登录页。

### 1.7 占位功能（本期不实现完整流程）

- [ ] 「忘记密码」点击后可展示「功能建设中」或跳转占位页，不阻塞主流程。
- [ ] 「企业微信」点击后可展示「功能建设中」或占位提示。
- [ ] 「简体中文」语言切换为占位入口，本期不强制实现 i18n 切换。

## 2. 接口验收

| 接口 | 方法 | 路径 | 验收要点 |
|---|---|---|---|
| 登录 | POST | `/api/v1/auth/login` | 参数校验；成功返回 token 与用户信息；401/403/429 错误码正确 |
| 当前用户 | GET | `/api/v1/auth/me` | 已登录返回用户信息；未登录或过期返回 401 |
| 退出登录 | POST | `/api/v1/auth/logout` | 成功返回 `{ "success": true }`；客户端清除登录态 |

错误码验收：

| HTTP 状态码 | 错误码 | 场景 |
|---|---|---|
| 400 | `AUTH_INVALID_REQUEST` | 参数缺失或格式错误 |
| 401 | `AUTH_INVALID_CREDENTIALS` | 账号或密码错误 |
| 403 | `AUTH_USER_DISABLED` | 用户被禁用 |
| 429 | `AUTH_TOO_MANY_ATTEMPTS` | 登录尝试过于频繁（若本期实现） |

## 3. 数据验收

- [ ] `users` 表包含：id、username、phone、email、password_hash、display_name、role、status、last_login_at、created_at、updated_at。
- [ ] 密码以哈希形式存储，数据库中无明文密码。
- [ ] 角色字段支持 `admin`、`employee`、`store_owner`。
- [ ] 用户状态字段支持 `active`、`disabled`。
- [ ] `login_logs` 表结构可预留，本期可不写入业务数据。

## 4. 技术验收

- [ ] 后端登录接口使用 Pydantic Schema 校验请求体。
- [ ] API 错误码与前端提示文案一致。
- [ ] 前端登录逻辑封装在 `src/web/src/features/auth/`，不散落在页面中。
- [ ] 前端不手写 Orval 生成目录代码。
- [ ] 新增接口后同步 `docs/03-api-index.md`。
- [ ] 新增表结构后同步 `docs/04-database-design.md`。
- [ ] 补充后端认证接口单元/集成测试。
- [ ] 补充前端登录表单与路由守卫测试。

## 5. UI 与交互验收

- [ ] 视觉风格与 `rules/ui-design.md` 及原型说明一致。
- [ ] 品牌金仅用于 Logo、主按钮、关键链接，避免滥用。
- [ ] 输入框、按钮圆角为 `2px–3px`，分割线使用低透明白色。
- [ ] 输入框 focus 态边框为品牌金弱强调。
- [ ] 错误提示克制，不使用大面积红色背景。
- [ ] 支持 Tab 键顺序：语言切换 → 用户名 → 密码 → 记住我 → 忘记密码 → 登录 → 企业微信。
- [ ] 表单字段具备可访问 label，错误提示关联 `aria-describedby`。

## 6. 异常场景验收

| 场景 | 前端表现 | 后端处理 |
|---|---|---|
| 账号为空 | 输入框下方提示「请输入用户名」 | 不请求接口 |
| 密码为空 | 输入框下方提示「请输入密码」 | 不请求接口 |
| 账号或密码错误 | Toast 或表单错误提示 | 返回 401 |
| 用户被禁用 | 提示「账号已停用，请联系管理员」 | 返回 403 |
| 网络异常 | 提示「网络异常，请稍后重试」 | 无 |
| 登录过期 | 跳转登录页，提示重新登录 | 返回 401 |
| 权限不足 | 展示无权限页面 | 返回 403 |

## 7. 本期不包含（不作为验收阻塞项）

- 店主账号自助注册。
- 忘记密码完整闭环。
- 手机验证码登录。
- 企业微信扫码登录。
- 微信小程序登录。
- 多租户组织体系。
- 细粒度 RBAC 权限配置页面。
- 登录审计报表。

## 8. 验收通过条件

满足以下全部条件视为本需求验收通过：

1. 第 1 节功能验收项全部通过（1.7 占位项按约定行为通过即可）。
2. 第 2 节接口验收通过。
3. 第 3 节数据验收通过。
4. 第 4 节技术验收通过。
5. 第 5 节 UI 与交互验收通过。
6. 第 6 节异常场景均可复现且表现符合预期。
