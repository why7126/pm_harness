# REQ-0001 用户登录 - Web 管理端

## 1. 背景

瓷砖信息管理平台包含店主端与企业内部管理端。管理端用于企业内部员工维护瓷砖信息、分类、规格、价格、图片素材与上下架状态，因此需要提供独立的 Web 管理端登录入口，确保内部员工能够安全进入后台。

本需求基于 `ui-design.md` 中「工业石材 · 暗色旗舰风」设计方案，优先落地 Web 管理端登录页。

## 2. 目标用户

- 企业内部运营员工：维护瓷砖基础资料、图片、规格参数、价格信息。
- 企业内部管理员：管理账号、权限、数据审核与系统配置。

## 3. 页面范围

| 项目 | 内容 |
|---|---|
| 页面名称 | Web 管理端登录页 |
| 页面路径建议 | `/admin/login` |
| 终端 | Web 桌面端优先，兼容移动 Web |
| 不包含 | 注册、找回密码完整流程、验证码真实发送、企业微信真实 OAuth |

## 4. 业务目标

- 建立企业内部员工进入管理端的统一入口。
- 通过品牌视觉强化平台专业、高端、工业化的数据管理定位。
- 为后续权限、账号体系、企业微信登录预留入口。

## 5. 核心用户故事

### US-001 员工账号密码登录

作为企业内部员工，我希望通过账号和密码登录管理端，以便维护瓷砖信息。

**验收标准：**

- 页面包含账号输入框、密码输入框、记住登录状态、忘记密码入口和登录按钮。
- 未输入账号或密码时，点击登录应出现字段级错误提示。
- 输入框聚焦时仅显示金色边框，不出现额外 ring-offset。
- 登录按钮为金色实底，文字为深色。

### US-002 企业微信登录

作为企业内部员工，我希望可以通过企业微信快速登录，以便减少账号密码输入。

**验收标准：**

- 页面提供「企业微信登录」入口。
- 入口使用绿色官方风格图标占位。
- 当前原型阶段点击后可 noop 或 toast 提示，不要求真实 OAuth。

### US-003 语言切换

作为管理端用户，我希望在登录页右上角看到语言切换入口，以便后续支持多语言。

**验收标准：**

- 右上角显示「简体中文」和 ChevronDown 图标。
- 当前原型阶段点击后不改变语言。

## 6. 信息架构

```text
登录页
├── 左侧品牌视觉区（桌面端显示）
│   ├── STONEX Logo
│   ├── 品牌主文案
│   ├── 材质数据看板
│   └── 石材 / 瓷砖纹理视觉
└── 右侧登录操作区
    ├── 语言切换
    ├── 登录标题
    ├── 账号输入框
    ├── 密码输入框
    ├── 记住我 / 忘记密码
    ├── 登录按钮
    ├── 第三方登录分割
    └── 企业微信登录
```

## 7. 字段与控件

| 控件 | 类型 | 必填 | 说明 |
|---|---|---:|---|
| 账号 | 文本输入 | 是 | 支持手机号、邮箱或员工账号 |
| 密码 | 密码输入 | 是 | 支持显示 / 隐藏密码扩展，本期可不实现 |
| 记住登录状态 | 复选框 | 否 | 默认不勾选 |
| 忘记密码 | 链接 | 否 | 后续跳转找回密码页 |
| 登录按钮 | 按钮 | 是 | 提交账号密码 |
| 企业微信登录 | 按钮 | 否 | 企业微信 OAuth 预留入口 |
| 语言切换 | 下拉入口 | 否 | 预留多语言能力 |

## 8. 交互规则

- 输入框默认透明底、强分割线边框。
- 输入框聚焦时边框切换为品牌金色。
- 登录按钮 hover 时可轻微提亮或上浮，但保持克制。
- 登录失败时建议展示字段级或按钮下方错误，不在表单上方增加 notice 横幅。
- 移动端隐藏左侧品牌视觉区，表单区域居中显示，最大宽度 520px。

## 9. 视觉约束

- 背景：右栏使用深色背景 `#18160F` / `bg-deep`。
- 品牌金：`#C8A055` 仅用于 Logo 强调、主 CTA、重点数据。
- 输入框桌面高度：64px；移动端高度：52px。
- 表单项间距：28px。
- 标题到表单距离：48px。
- 主按钮到第三方登录区域距离：56px。
- 圆角接近直角：2px～3px。
- 禁止在表单上方放置 notice 横幅。

## 10. 非功能需求

- 首屏在 1440px 桌面宽度下无需滚动。
- 页面需支持键盘 Tab 聚焦。
- 文本对比度需满足暗色背景下可读性。
- HTML 原型需尽可能与 PNG 原型一致，方便 Cursor / 前端开发直接参考。

## 11. 待确认事项

- 账号体系是否使用手机号、邮箱、员工号三者全部支持。
- 是否强制接入企业微信单点登录。
- 登录失败次数限制、验证码、人机验证是否进入后续需求。
- 管理员与普通员工是否在登录后进入不同首页。
