## US-001 企业内部员工登录管理端

**作为** 企业内部员工，  
**我希望** 使用公司分配的账号和密码登录管理端，  
**以便** 维护瓷砖产品资料、图片、规格、分类与上下架状态。

### 验收要点

- 可通过 `/admin/login` 打开登录页，页面布局与 `prototype/web/user-login.png` 一致。
- 输入正确账号密码后，进入 `/admin/dashboard`。
- 输入错误账号或密码时，不进入系统，并提示「账号或密码错误」。
- 未登录直接访问管理端页面时，自动跳转登录页。
- 登录过程中按钮展示 loading，防止重复提交。
- 勾选「记住我」后，刷新页面仍保持登录状态。
- 可通过管理端顶部菜单退出登录，退出后返回登录页。

### 关联功能

- FR-001 ~ FR-005：账号密码登录
- FR-009 ~ FR-011：退出登录
- FR-012 ~ FR-014：路由守卫

## US-002 系统管理员登录管理端

**作为** 系统管理员，  
**我希望** 登录后进入管理端并拥有更高权限标识，  
**以便** 后续管理用户、角色配置与系统设置。

### 验收要点

- 登录成功后后端返回用户角色为 `admin`。
- 前端可根据角色决定导航菜单与页面权限。
- 非管理员访问管理员专属页面时应被拒绝，展示无权限提示。
- 被禁用管理员账号不能登录，提示「账号已停用，请联系管理员」。

### 关联功能

- FR-006 ~ FR-008：当前用户信息
- FR-015：权限不足处理

## US-003 店主端后续登录扩展（本期预留）

**作为** 瓷砖零售店店主，  
**我希望** 后续可以在 Web 或小程序登录，  
**以便** 查看与我相关的瓷砖资料、收藏和询价记录。

### 验收要点（本期）

- 数据模型预留 `store_owner` 角色。
- 登录接口与认证架构不阻碍后续小程序登录接入。
- Web 展示端 `/login` 本期可不开放；原型中的统一登录页设计可复用于后续店主端。
- `store_owner` 若误用管理端账号登录，应拒绝访问管理端受保护页面。

### 本期不实现

- Web 展示端店主登录入口。
- 微信小程序登录。
- 收藏、询价等个性化能力。

## 跨角色通用期望

| 期望 | 说明 |
|---|---|
| 统一错误提示 | 登录失败不区分「用户不存在」与「密码错误」 |
| 安全存储 | 密码不得明文存储或传输至日志 |
| 登录态恢复 | Web 端刷新后可恢复登录状态 |
| 过期处理 | Token 过期时跳转登录页并提示重新登录 |
| 占位入口 | 「忘记密码」「企业微信」可展示但本期可不实现完整流程 |
