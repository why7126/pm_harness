# Web 管理端登录页 - 产品原型图上下文工程

## 1. 原型目标

本原型用于指导 Cursor / AI 前端开发将 PNG 视觉稿还原为 HTML 页面。开发时应优先参考本上下文文档和 `login.html`，而不是仅凭 PNG 图片理解页面。

## 2. 设计来源

- 设计规范：`ui-design.md`
- 页面：Web 管理端登录页
- 视觉风格：工业石材 · 暗色旗舰风
- 关键约束：暗色背景、克制金色、极细分割线、近直角圆角、表单专业感。

## 3. 页面画布

| 项目 | 值 |
|---|---|
| 推荐设计稿尺寸 | 1440 × 1024 |
| 桌面布局 | 左右 50% 分屏 |
| 左栏 | 品牌视觉与材质展示 |
| 右栏 | 登录表单 |
| 移动布局 | 隐藏左栏，表单全屏居中，最大宽度 520px |

## 4. 页面结构说明

```text
<body class="login-page">
  <main class="login-shell">
    <section class="brand-panel">...</section>
    <section class="form-panel">...</section>
  </main>
</body>
```

### 4.1 左侧品牌视觉区

左侧占 50% 宽度，用于建立品牌和材质感。包含：

1. 顶部品牌 Logo：`STONEX`，字母间距较大，金色显示。
2. 中部主标题：强调「瓷砖信息管理平台」。
3. 副文案：说明平台用于企业内部瓷砖数据维护。
4. 材质统计卡片：展示瓷砖 SKU、纹理图、规格数据、门店覆盖等。
5. 右下角材质样板拼贴：通过渐变模拟石材和瓷砖纹理。

### 4.2 右侧登录表单区

右侧占 50% 宽度，背景深色。包含：

1. 右上角语言切换：「简体中文⌄」。
2. 表单容器：最大宽度约 420px。
3. 眉标：`ADMIN PORTAL`。
4. 标题：`登录管理端`。
5. 描述：说明仅限企业内部员工使用。
6. 账号输入框。
7. 密码输入框。
8. 记住登录状态 + 忘记密码。
9. 登录按钮。
10. 第三方登录分割线。
11. 企业微信登录按钮。
12. 底部安全说明。

## 7. 组件状态

### 输入框

- 默认：透明背景，`1px solid rgba(255,255,255,0.1)`。
- 聚焦：`border-color: #C8A055`，无额外阴影，无 ring-offset。
- Placeholder：`rgba(237,232,223,0.25)`。
- 文本：`#EDE8DF`。

### 主按钮

- 背景：`#C8A055`。
- 文字：`#18160F`。
- 圆角：`2px`。
- Hover：亮度略提升，`transform: translateY(-1px)`。

### 复选框

- 尺寸：18px。
- 默认：透明底，白色弱边框。
- 选中：金色填充。

### 企业微信按钮

- 背景：透明。
- 边框：`rgba(255,255,255,0.1)`。
- 图标：绿色官方风格圆形 SVG 占位。
- 文字：`#EDE8DF`。

## 9. 开发还原建议

- React / Next.js 中可拆为 `LoginPage`、`BrandPanel`、`LoginForm`、`LanguageSwitch`、`WeComButton`。
- Web 端正式实现必须替换为 semantic token class，例如 `bg-page`、`text-brand-gold`。
- 本 HTML 为静态原型，允许保留 CSS 变量；生产代码需接入项目 Design Token。
- 左侧材质图在正式项目中替换为 `/images/login-material-showcase.jpg`。
- 企微图标在正式项目中替换为 `/icons/wecom.svg`。

## 10. 与 PNG 一致性检查清单

- [ ] 是否为桌面左右 50% 分屏。
- [ ] 左栏是否为品牌视觉，右栏是否为登录区。
- [ ] Logo 是否使用金色且字距较大。
- [ ] 输入框高度是否接近 64px。
- [ ] 表单项间距是否接近 28px。
- [ ] 主按钮是否为金色实底。
- [ ] 是否没有表单上方 notice 横幅。
- [ ] 语言切换是否位于右上角。
- [ ] 企业微信入口是否位于第三方登录区域。
- [ ] 移动端是否隐藏左栏。
